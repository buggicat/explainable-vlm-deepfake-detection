# Explainable and Bias-Free: Benchmarking Commercial VLMs for Transparent Deepfake Detection

 Evaluates commercial vision-language models (GPT, Gemini, Claude) on deepfake detection under two scenarios — a standard biased benchmark and a content-matched bias-free benchmark — across two conditions: visual-only and agentic (model uses its own code execution and web search).

## Repository Structure

## Datasets

### Scenario 1 — RVF10K (real vs GAN fakes)

Download the **RVF10K** dataset from Kaggle and place the `valid/` split under `data/rvf10k/valid/`:

```
data/rvf10k/valid/
  real/   ← 1,500 images used in this study
  fake/   ← 1,500 images used in this study
```

> Kunichetty, S. (2023). *RVF10K* [Dataset]. Kaggle. https://www.kaggle.com/datasets/sachchitkunichetty/rvf10k

Only the `valid/` split is used. The `train/` split is not needed.

### Scenario 2 — SDXL-generated bias-free pairs (our contribution)

Our generated images and raw evaluation results are hosted on Zenodo: https://doi.org/10.5281/zenodo.21009932

Download and extract into `data/scenario2/`:

```
data/scenario2/
  self_cond/       ← 1,500 SDXL self-conditioned fakes
  local_inpaint/   ← 1,500 SDXL locally inpainted fakes
  real_1024/       ← 1,500 real faces resized to 1024px (SDXL input)
```

The evaluation manifest linking all images is already included at `data/eval_manifest.json`.

## Pipeline

Run the numbered scripts in order:

### 1. Generate Scenario 2 bias-free pairs *(skip if downloading from Zenodo)*

Our pre-generated Scenario 2 images are available on Zenodo (see **Datasets** section above). If you want to regenerate them from scratch, you need a GPU with ≥16 GB VRAM:

```bash
python scripts/1_generate_bias_free_pairs.py \
  --rvf10k-dir data/rvf10k/valid/real \
  --out-dir data/scenario2 \
  --mode both
```

Generates content-matched real/fake pairs from the RVF10K real faces using Stable Diffusion XL (self-conditioned reconstruction + local inpainting).

### 2. Build evaluation manifest

> **Skip if:** `data/eval_manifest.json` already exists (included in this repo).

```bash
python scripts/2_build_eval_manifest.py \
  --rvf10k-dir data/rvf10k/valid \
  --scenario2-dir data/scenario2 \
  --out data/eval_manifest.json
```

### 3. Build forensics reference card

> **Skip if:** `prompts/techniques_reference.md` already exists (included in this repo).

```bash
python scripts/3_build_techniques_reference.py \
  --taxonomy data/multimedia_forensics_techniques.txt \
  --chunks   data/chunks.jsonl \
  --out      prompts/techniques_reference.md
```

### 4. Run VLM evaluation

> **Skip if:** you downloaded `vlm_eval_results.zip` from Zenodo and extracted `vlm_eval_raw.jsonl` to `results/`.

```bash
# Smoke test — 10 images, all models, both conditions
python scripts/4_run_vlm_eval.py \
  --manifest data/eval_manifest.json \
  --models openai gemini anthropic \
  --conditions A B \
  --limit 10 \
  --out results/vlm_eval_raw.jsonl

# Full run (resumable — re-running skips completed records)
python scripts/4_run_vlm_eval.py \
  --manifest data/eval_manifest.json \
  --models openai gemini anthropic \
  --conditions A B \
  --provider-limits openai=1200 gemini=1200 anthropic=200 \
  --out results/vlm_eval_raw.jsonl
```

**Condition A** = visual-only (no tools). **Condition B** = agentic (model uses its own code execution sandbox and web search).

### 5. Compute metrics

> **Skip if:** you downloaded `vlm_eval_results.zip` from Zenodo and extracted `vlm_eval_scored.jsonl` to `results/`. The pre-computed summary is already included as `results/vlm_eval_scored.summary.json`.

```bash
python scripts/5_compute_metrics.py \
  --in  results/vlm_eval_raw.jsonl \
  --out results/vlm_eval_scored.jsonl
```

Outputs `results/vlm_eval_scored.summary.json` with accuracy, F1, AUC, bias gap, XAI scores, and calibration metrics.

### 6. Run LLM-as-judge explanation audit

> **Skip if:** `results/explanation_audit.jsonl` and `results/explanation_audit_summary.json` already exist (included in this repo).

```bash
python scripts/6_explanations_llm_judge.py \
  --in      results/vlm_eval_scored.jsonl \
  --out     results/explanation_audit.jsonl \
  --summary results/explanation_audit_summary.json \
  --sample 200 \
  --providers openai gemini \
  --condition B
```

Uses Claude as an independent judge to audit groundedness of agentic explanations.

### 7. Export paper tables

> **Skip if:** the `.txt` table files already exist in `metric_outputs/` (included in this repo).

```bash
python scripts/7_export_metric_tables.py \
  --results-dir results \
  --figures-dir metric_outputs
```

Writes all paper tables as `.txt` files to `metric_outputs/`.

## Experimental Design

- **Models:** GPT (gpt-5.4-mini), Gemini (gemini-2.5-pro), Claude (claude-sonnet-4-6)
- **Scenario 1:** Standard RVF10K — 1,500 real + 1,500 GAN fake faces (biased benchmark)
- **Scenario 2:** Content-matched pairs — fakes generated from the same 1,500 real faces via SDXL (bias-free benchmark)
- **Condition A:** Visual-only — model judges from visual perception alone
- **Condition B:** Agentic — model uses its own native code execution and web search
- **Headline metric:** Scenario 1 accuracy − Scenario 2 accuracy = bias reliance gap
